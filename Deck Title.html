Choosing an org
- Based on your interest
- Experience required
- Active ?



Talking to developers
Slack, Gitter, IRC, Skype, Hangouts
Introductory Mail
      Using IRC

- Say Hi ! & ask the question 
- Wait for somebody to answer
- Learn how to use IRC
- Join the mailing-list/Google-groups 
- Talk often and show enthusiasm
- Be patient, most of the devs have a
  day job and cannot answer your 
  questions immediately 
- It's rude to DM anyone on IRC 
   without their express permission(!)
- Use proper grammar, no slangs (!)
Using mailing list
Ask queries, file bugs and there you go !!
Contribute some code
- Project setup
- Find beginner friendly bug in the
   issue tracker
- Ask the devs if you can't find one
- Write some code fixing the issue
- Send a PR or propose a patch.
- Get it reviewed
- Get it merged !! 

Beginner Bugs 

**Submit a proposal**
The Idea !!
Look for an existing idea on the ideas page or purpose 
a new one by talking to the devs

- Research first, ask questions later
- Assumptions are the hardest thing
  to refactor
- Handle disagreements gracefully
- Study the existing code 
- Be practical, write some code
- Discuss thoroughly with devs (!)
- Choose submission time wisely


Keep contributing and
                   wait ;)
